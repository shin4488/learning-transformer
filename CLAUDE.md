# CLAUDE.md

Transformer解説書『ゼロから理解するTransformer』のリポジトリ。初心者向けの日本語技術書(全16章 + README)を Markdown で管理し、VitePress で静的サイトとして公開している。

## 構成

- `README.md`, `01-*.md` 〜 `16-*.md`: 書籍本体。**GitHub 上でもそのまま読める記法を維持する**(下記の記法規約を参照)
- `privacy-policy.md`: AdSense 用のプライバシーポリシー(サイトにのみ意味を持つページ)
- `.vitepress/config.mts`: サイト設定。サイドバー・SEO(ページ別 description)・数式記法のビルド時変換・AdSense/GA の ID 設定口(`ADSENSE_CLIENT` / `GA_ID` / `SITE_URL`)がここにある
- `DEPLOY.md`: 運用手順(ホスティング・ドメイン・AdSense/GA)。サイトには公開されない(srcExclude)
- `.github/workflows/deploy.yml`: GitHub Pages への自動デプロイ(`DOCS_BASE=/learning-transformer/` を設定してビルド)。Cloudflare Pages では環境変数なしでビルドする(両対応)

## コマンド

```sh
npm run dev      # 開発サーバ(要 Node 20)
npm run build    # 本番ビルド → .vitepress/dist
docker compose up             # Docker で開発サーバ(http://localhost:5173)
docker compose run --rm build # Docker で本番ビルド
```

## 記法規約(重要)

本文の md は GitHub と VitePress の両方で正しく表示される必要がある。**GitHub の Markdown 処理は数式の中身を壊すことがある**ため、以下を厳守する(VitePress 側はビルド時変換で吸収している)。

### 数式

- **インライン数式**: 基本は `$ ... $`。ただし**数式と日本語文字の間には必ず半角空白**を入れる(密着すると GitHub で描画されない)
- 内容に `_x`(波括弧なし下付き)が2つ以上ある・`\\` を含む・両端が括弧に接するなど、GitHub で壊れるインライン数式は **`` $`...`$ ``(コード保護形式)** を使う
- **ブロック数式**: `\\`(行列の行区切り)を含むものは `$$` ではなく **```` ```math ```` フェンス**を使う(`$$` 内の `\\` は GitHub が食う)。`\\` を含まない単純な式は `$$ ... $$` でよい
- 使用禁止の記法: `\operatorname`(→ `\arg\max` 等)、`\{` `\}`(→ `\left( \right)`)、`\|`(→ `\lVert \rVert`)、`\%`(→ 比率は数値で書き % は本文へ)、`\\[2mm]` などの行間指定
- ブロック数式の直後には「**読み下し**: …」(日本語での読み方)を付ける

### 太字

- `**...**` の内側の端が括弧などの約物で、外側が文字と密着する場合は `**` の外側に半角空白を入れる(例: `**トークン化(tokenization)** の方法`)。守らないと GitHub で太字にならない

### 図

- 構成図・フロー図: Mermaid `graph TD` / `flowchart TD`(向きは上から下に統一)。ノードラベルは `["..."]` とダブルクォートで囲む
- 関数グラフ・棒グラフ: Mermaid `xychart-beta`(値は必ず正しく計算する。凡例が無いのでキャプションで系列を説明)
- `quadrantChart` は GitHub でエラーになるため**使用禁止**
- ベクトルの幾何・行列レイアウトなど Mermaid で表現できない図のみ ```` ```text ```` の ASCII 図。**描画部分は半角文字のみ、日本語ラベルは行末か独立行のみ**(全角文字が描画の途中にあると桁がずれる)
- スタイル指定には必ず `color:#24292f` など明示的な文字色を付ける(ダークモード対策)

### 引用・注記

- `>` の素の引用は実際の出典引用のみ。補足・注意は GitHub Alert(`> [!NOTE]` `> [!TIP]` `> [!IMPORTANT]` `> [!WARNING]`)を使う

## 文章のスタイル

- です・ます調の講義風。読者は「LLM・機械学習・数学に詳しくない人」(前提は文系高校数学)
- **AIっぽい表現を避ける**: 「心臓部」「〜の旅」「魔法」「革命」「驚くべき」「圧倒的」「処方箋」「〜が宿る」「親戚」「玉突き」「沼」のような大げさな比喩・翻訳調は使わない。名詞の「直感」は「考え方」「イメージ」等にする。2連ダッシュ「——」は使わない
- 章構成の約束: 前の章までに定義した用語だけを使う(芋蔓式)。章冒頭に「この章で学ぶこと」「この章の前提」、章末に「この章のまとめ」「次の章へ」
- 通し例は「猫は魚が好き」(トークン列 `[猫, は, 魚, が, 好き]`)
- 章タイトル(h1)は他ファイルから参照されているため変更しない

## 検証

- 修正後は GitHub の Markdown API(`https://api.github.com/markdown`)にファイルを POST し、`math-renderer` 以外に生の `$` が残っていないかを確認すると、数式の描画崩れを機械的に検出できる
- `npm run build` が通ること(リンク切れがあるとビルドが落ちる)
